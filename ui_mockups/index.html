Calorie Counter Application

UI Mockup Screens - Interactive Prototypes

About These Mockups

These are fully functional HTML prototypes of the Calorie Counter application screens, built according to Material Design 3 specifications. Each screen is standalone and includes interactive elements, responsive design, and accessibility features.

Design System
Material Design 3
Responsive
Mobile, Tablet, Desktop
Accessibility
WCAG 2.1 Compliant
Technology
HTML5, CSS3, JavaScript

Landing/Search Screen

The main entry point where users can search for food items to get nutritional information.

  • Material Design search input with validation
  • USDA attribution and branding
  • Responsive layout for all devices
  • Form validation and error handling
  • Keyboard navigation support

Search Results Screen

Displays search results in a clean, scannable list format with nutritional information.

  • Persistent search header for easy re-searching
  • Three-column result layout (food, portion, calories)
  • Load more functionality with pagination
  • Responsive grid layout for different screen sizes
  • Interactive result selection

No Results Screen

Helpful empty state with suggestions and popular searches when no results are found.

  • Empty state illustration with clear messaging
  • Actionable suggestions for better search results
  • Popular search tags for quick access
  • Try again functionality
  • Contextual help and guidance

Loading Screen

Loading state with progress indication and cancel option for better user experience.

  • Animated loading spinner with Material Design styling
  • Dynamic loading messages that update over time
  • Cancel search option after 3 seconds
  • Accessibility announcements for screen readers
  • Simulated search completion with realistic timing

Navigation Flow

The screens are interconnected and simulate a real application flow:

  • 1. Start at Landing/Search to enter a search term
  • 2. View Loading screen during search processing
  • 3. See Search Results for successful searches
  • 4. View No Results when no matches are found
